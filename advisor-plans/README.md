### ⚠️ Execution incident & recovery (read before merging)

The six plans were dispatched as parallel executors with `isolation: "worktree"`.
**That isolation silently failed** — all five initial executors shared the *main*
working tree's single HEAD and stepped on each other (checkouts, cherry-picks,
resets). Recovery:

- Git commits are immutable, so each plan's work survived as identifiable commits.
  The reviewer rebuilt clean, isolated per-plan branches by cherry-picking each
  plan's commit(s) onto `main`, then independently re-verified (typecheck + tests +
  diff audit) — the results above are from that re-verification, not the executors'
  (untrusted) self-reports.
- **005 (Sentry)** was the only plan *lost* in the collision (no commit survived); it
  was re-run cleanly in a dedicated worktree.
- **007** was run afterward in a dedicated manual worktree on top of recovered 006.
- The full collision state is preserved under tag **`salvage/collision-2026-06-21`**
  in case anything needs archaeology.
- **Root cause for next time:** do not rely on `isolation: "worktree"` here; create
  worktrees manually (`git worktree add <dir> <base>`) and pin each agent to its dir
  with `git -C`. Even so, executors tended to run `git checkout -b` despite
  instructions — harmless only because each had its own physical worktree.

## Findings considered and rejected (do not re-audit)

- **"Negative-index crash" in exercise nav** (`SessionView.tsx:1033`) — guard is
  `<= 0`, and `-1 <= 0`, so the `exerciseBoundaries[-2]` path is unreachable.
- **"Rest-timer adjustment race"** (`SessionView.tsx:152-171`) — single-threaded JS;
  `adjustTime` completes before any `setInterval` tick. No interleaving.
- **"`currentStep` null crash in `handleLogged`"** — UI-guarded by
  `phase === 'exercise' && currentStep` (line 1273); not reachable.
- **"Committed build logs"** (`apps/mobile/*.log`) — not tracked, gitignored.
- **"Firebase admin key / `google-services.json` committed"** — not tracked, never in
  history, gitignored (root `.gitignore` 70–75). Hygiene only, not a leak.
- **Hardcoded Sentry DSN, raw PB error strings to UI, token in AsyncStorage,
  `(err as Error)` casts** — real but low-value / idiomatic.
- **`@xmldom/xmldom` / `uuid` advisories** — dev-only build tooling, not shipped.
