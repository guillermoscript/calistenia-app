## Findings considered and rejected (do not re-audit)

- **Stale closure on mutation `key`/`limit`** (useReactions, useNotifications, useLeaderboard): rejected. React Query v5 `useMutation` re-applies options every render, so `onMutate`/`onError` use the latest render's closure — not a mount-time snapshot. The only real (narrow) variant is an in-flight mutation whose key changes before `onError`; that is folded into plan 010 (carry the resolved key in mutation context).
- **useBodyMeasurements double `Date.now()` ghost entry**: rejected. `onSuccess` reconciles by `ctx.optimisticId`, which always exists in the cache (online or offline), so no ghost/duplicate results. (Minor: the redundant id-gen in `mutationFn` is cosmetic.)
- **useChallenges `expiredIds` effect "loop"**: rejected. Marked-ended challenges drop out of `expiredIds` on the next fetch, so the effect self-terminates after one cycle; the writes are idempotent. Worst case is one redundant write, not a loop.
- **No AbortSignal passed to queryFns**: rejected. The hooks deliberately set `$autoCancel: false` on PocketBase calls; request cancellation is an intentional design choice, not an oversight.
- **useProgress sets_log "not program-scoped"**: rejected as a bug. Sets are fetched user-wide on purpose — exercise history and PRs are global (a pull-up PR is a pull-up PR). The only real cost is a redundant refetch of all sets on program switch, which is subsumed by the perf work, not a correctness defect.

## Minor findings noted but not planned

- `useReferralPoints.getTransactions(limit)` ignores its `limit` arg (pinned to 50). Known/accepted — the only caller uses the default.
- `useNotifications` realtime subscription only invalidates on `create` events; `update`/`delete` (multi-device read-state) are ignored. Low impact on single-device use; `markAsRead` already updates locally.
- `usePrograms.refreshPrograms` uses a raw `['programs','detail']` array literal instead of the `qk` factory. Works today via React Query prefix-matching; fragile if the key shape changes. Cosmetic.
